# ArtPipeline 目录说明

## source/ — 原始生成物

ComfyUI 或**云端 API**（万相 / 混元 / 即梦 / Stability）的**直接输出**，可按版本随意命名。

| 子目录 | 内容 | 建议导出尺寸 |
|--------|------|----------------|
| `source/roles/` | 职业 / 背叛者半身头像 | **512×512** 方图，真透明底 |
| `source/items/` | 道具 icon（自带暗金框） | **256×256** 方图 |
| `source/ui/` | 血量心、枪膛槽、徽标等 | 见 manifest（如 128×128） |

不必与 Unity 文件名一致；挑选后再放入 `inbox/`。

## inbox/ — 待入库

**文件名必须与 Unity 资源一致**（见 `manifest/icons.yaml`）。

| 子目录 | 正式文件名示例 |
|--------|----------------|
| `inbox/roles/` | `role_warrior.png`, `role_traitor.png` |
| `inbox/items/` | `item_amulet.png`, `item_spyglass.png` |
| `inbox/ui/` | `hp_heart_full.png`, `hp_heart_empty.png` |

运行 `generate_icons_comfyui.py --to-inbox --deploy` 会从这里复制到 `Assets/Resources/UI/Icons/`（无裁切/抠图后处理）。

## comfyui/workflows/

存放 **API Format** 导出的工作流 JSON。脚本 `generate_icons_comfyui.py`（待接）从此目录加载。

默认 ComfyUI 地址：`http://127.0.0.1:8188`（可在环境变量 `COMFYUI_URL` 覆盖）。

## tools/cloud/

云生图**模型注册表**（`registry.json`）与后续 API 实现。与 `pipeline_core.py`（ComfyUI）并行，checkpoint 以 `cloud:` 前缀区分。说明见 [cloud-generation.md](cloud-generation.md)。

## manifest/

机器可读的资产清单，与 `icon_ai_specs.py` 同步，便于 ComfyUI 批处理与人工核对。
