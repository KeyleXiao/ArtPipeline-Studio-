# ArtPipeline tools · 维护指南

## 日常流程

```
ComfyUI 运行
    ↓ artTool_ui / cli 生成
source/<分类>/          原始输出
    ↓ 复制（生成时可选自动）
inbox/<分类>/           待入库（文件名 = Unity 文件名）
    ↓ 导出
Assets/Art/...         Unity 贴图（UiArtCatalog 索引）
```

## 界面功能对照

| 功能 | 位置 | 说明 |
|------|------|------|
| 检测 ComfyUI | 工具栏「检测连接」 | 验证 Comflowy/ComfyUI 是否在线 |
| 刷新模型 | 工具栏「刷新模型」 | 扫描 ComfyUI `/models/checkpoints` 并更新下拉 |
| Checkpoint | 全局设置 + 分类设置 | 各分类可指定不同模型；空=用全局默认 |
| 新建分类 | 工具栏 / 文件 | 同步创建 source、inbox 目录，写入 config |
| 新建资源 | 工具栏 | 创建 asset 条目 + 复制默认工作流 JSON |
| 分类路径 | 基本信息 → 路径映射 | 修改后保存 → 写入 `pipeline_config.json` |
| 提示词 | 提示词页签 | 每张图独立 positive / negative |
| 工作流 | 工作流 JSON 页签 | 粘贴 ComfyUI API JSON，支持占位符 |
| 单个生成 | 选中资源 → 生成选中 | source + inbox |
| 批量生成 | 生成本类全部 | 当前分类所有 enabled 资源 |
| 导出 | 导出选中 / 导出全部 | inbox → Unity（不裁切） |

## 分类与 Unity 路径

默认映射（可在 UI 修改）：

| 分类 ID | source | inbox | Unity |
|---------|--------|-------|-------|
| roles | `ArtPipeline/source/roles` | `ArtPipeline/inbox/roles` | `Assets/Art/UI/Icons/Roles` |
| items | `ArtPipeline/source/items` | `ArtPipeline/inbox/items` | `Assets/Art/UI/Icons/Items` |
| skills | `ArtPipeline/source/skills` | `ArtPipeline/inbox/skills` | `Assets/Art/UI/Icons/Skills` |
| ui_status | `ArtPipeline/source/ui_status` | `ArtPipeline/inbox/ui_status` | `Assets/Art/UI/Icons/UI/Status` |
| backgrounds | `ArtPipeline/source/backgrounds` | `ArtPipeline/inbox/backgrounds` | `Assets/Art/UI` |

新增分类示例：ID=`fx`，Unity 路径填 `Assets/Art/UI/Icons/FX`。

## 修改配置的方式

1. **推荐**：`artTool_ui.py` 编辑后点各页签「保存」  
2. **高级**：直接编辑 `pipeline_config.json`（保存后 UI 点「重新加载配置」）

## 命令行等价操作

```bash
# 列出全部
python3 ArtPipeline/tools/cli.py --list

# 生成角色并导出 Unity
python3 ArtPipeline/tools/cli.py --category roles --to-inbox --deploy

# 单张
python3 ArtPipeline/tools/cli.py --file role_warrior.png --to-inbox --deploy
```

## 工作流 per-asset 策略

- 新建资源时自动从 `_default_sdxl_api.json` 复制到 `workflows/assets/<id>.json`
- 某张图需要特殊节点（如 ControlNet、LoRA）时，只改该 JSON，不影响其他图
- 占位符 `{{POSITIVE}}` 等由 `workflow_engine.py` 在提交 ComfyUI 前替换

## 易漏事项

- **ComfyUI 必须先启动**（Comflowy 或 `python main.py`）
- **checkpoint 名称**须与 `models/checkpoints/` 内文件名一致
- 角色默认 **1024 生成 → 512 入库**（`gen_scale: 2.0`）
- **不要默认开 rembg**（易裁掉手臂）；黑底方案靠 UI `coverFrame` 显示
- 修改 `pipeline_config.json` 后 Unity 侧只需替换 PNG，**不必改 C#**
- 旧 `manifest/icons.yaml` 仍作人类可读清单，**运行时以 pipeline_config.json 为准**

## 故障排查

| 现象 | 处理 |
|------|------|
| Load failed undefined（Comflowy 点模型） | 模型文件正常时可忽略；用 API/本工具生成 |
| 生成无输出 | 检查工作流 JSON 是否含 SaveImage 节点 |
| inbox 无文件 | 确认生成成功；看日志区报错 |
| Unity 不更新 | 点「导出」；Unity 重进 Play 或 Reimport PNG |
